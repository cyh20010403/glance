# 回眸（Glance）项目 AI 开发规则

# 项目简介

《回眸》是一款基于现实场景的匿名双向心动社交 App。

核心理念：

“所有故事，都始于一次回眸。”

产品强调：

* 青春感
* 克制感
* 情绪价值
* 现实中的瞬间连接

禁止将产品做成：

* 陌陌
* 探探
* 荷尔蒙社交
* 低俗约会软件

---

# 技术栈

## 前端

* Flutter
* Riverpod
* GoRouter
* Dio
* Freezed
* Clean Architecture
* MVVM

## 后端

* Spring Boot
* MySQL
* Redis
* WebSocket

---

# 开发规则

1. 所有代码必须可运行
2. 禁止生成伪代码
3. 所有文件必须标注路径
4. 所有功能必须模块化
5. 不允许破坏已有架构
6. 保持代码风格统一
7. 所有页面必须响应式
8. 所有状态管理统一使用 Riverpod
9. 所有网络请求统一封装
10. 所有 API 必须统一规范

---

# Flutter 架构规范

使用：

* Feature First
* Clean Architecture
* MVVM

目录结构：

lib/
├── core/
├── features/
├── shared/
├── routes/
├── theme/

---

# UI 风格规范

整体风格：

* iOS 极简风
* 白色背景
* 柔和渐变
* 青春感
* 呼吸感
* 轻量动画

禁止：

* 复杂渐变
* 霓虹风
* 赌博/夜店风
* 低俗社交 UI

---

# 页面设计规范

* 圆角统一
* 阴影轻量
* 动画克制
* 字体层级清晰
* 保持留白

---

# 输出规范

每次生成代码时：

必须输出：

1. 功能分析
2. 文件结构
3. 完整代码
4. 运行说明
5. 下一步建议

禁止一次性生成整个项目。

每次仅完成一个模块。

---

# 当前 MVP 功能

当前仅实现：

* 登录注册
* 附近发现
* 心动卡片
* 双向匹配
* 匿名聊天

禁止提前生成：

* 直播
* 视频聊天
* 动态广场
* 复杂推荐系统

---

# 产品核心

《回眸》的核心不是：

“附近的人”

而是：

“刚刚与你产生连接的人。”
